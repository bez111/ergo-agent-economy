Agent economy thesis

Autonomous agents need more than payment wrappers.

They need machine-readable terms, bounded authority, work verification and receipts that other systems can audit. Accord gives those pieces a shared protocol surface.

01

Terms before value

Price, asset, recipient, deadline, refund rule, task definition and verifier belong in the agreement — not in a side-channel.

02

Bounded authority

Agents should spend through purpose-bound limits, Notes, policies and expiries instead of unrestricted wallets.

03

Verification before settlement

A payment receipt says value moved. A verification receipt says whether the promised work was actually accepted.

04

Receipts as infrastructure

Receipts are the basis of accounting, dispute handling, reputation, replay checks and conformance.

Protocol architecture

From paid request to verifiable economic record.

  1. 01

    Agree

    Buyer and seller form a canonical Accord Agreement and deterministically hash it.

  2. 02

    Authorize

    A payment proof, Note, escrow transaction or mandate references the agreement.

  3. 03

    Execute

    The agent, API, MCP tool or service performs the work under explicit constraints.

  4. 04

    Verify

    A verifier evaluates output and emits a signed Verification Receipt.

  5. 05

    Settle

    The rail completes the economic side and produces a Settlement Receipt.

Small vocabulary, large surface

Three objects carry the work contract.

Accord keeps the protocol legible: every integration can inspect what was promised, how acceptance was decided and what settled.

A

Accord Agreement

Defines task, parties, price, asset, deadline, verifier, rail assumptions and acceptance criteria.

V

Verification Receipt

Records accepted, rejected or partially accepted work with verifier identity and task evidence.

S

Settlement Receipt

Captures the economic proof: rail, transaction, payment proof, accounting state or redemption event.

Rail-agnostic by design

Accord separates work semantics from settlement rails.

Use the same agreement and receipt model across HTTP payments, MCP tools, programmable Notes and future agent-payment networks.

FAQ

Answers that should survive copy, search and agent summarization.

What does Accord add to MCP?

MCP explains how agents call tools. Accord adds paid-tool terms, completion verification and settlement receipts around those calls.

What does Accord add to x402?

x402 is strong for HTTP payment challenges. Accord adds the work agreement and verifier receipt so a paid request can prove what was completed.

Why is Ergo the first reference programmable-settlement rail?

Ergo's eUTXO model, ErgoScript, native tokens, Babel-fee-style patterns and PoW settlement make it a strong fit for programmable testnet settlement.

Can builders use Accord with real funds today?

No. Current Accord v0 rail implementations are testnet-first and not production-certified. Real-fund use is blocked until signed audit manifests allow it.
