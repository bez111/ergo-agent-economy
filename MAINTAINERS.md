# Maintainers

Accord Protocol is currently in v0 and maintained through a maintainer-led public RFC process.

## Maintainer areas

| Area | Scope |
|---|---|
| Core protocol | Accord Agreement, Verification Receipt, Settlement Receipt |
| Schemas and specs | `specs/`, `schemas/`, object versioning |
| Conformance | `@accord-protocol/conformance`, test vectors, certification levels |
| Gateway / 402 | Accord/402 HTTP middleware and replay protection |
| MCP | Accord/MCP wrappers and paid tool flow |
| Ergo rail | Note / Reserve / Tracker / Acceptance Predicate reference rail |
| Other rails | Rosen, Base/EVM, x402-compatible adapters |
| Security and audits | Security policy, audit manifests, mainnet certification gates |
| Release engineering | npm, PyPI, GitHub releases, changelog |

## Becoming a maintainer

The v0 process is intentionally lightweight:

1. Ship useful changes.
2. Review related PRs.
3. Demonstrate judgment around status, security, and compatibility.
4. Ask to be added to this file.

A formal Technical Steering Group may replace this process when Accord reaches v1.
